文件的打开与关闭

发表于 2019-10-08 | | 本文总阅读量 次

文件的打开与关闭操作在王道考研书上和操作系统教材上都写得比较简略，我没太看明白，于是找来modern os的原版书研究一下。

首先看一下书中对打开操作的定义

Open: Before using a file, a process must open it. The purpose of the open call is to allow the system to fetch the attributes and list of disk address into main memory for rapid access on later calls.

意思就是在对文件进行其他的操作之前，先将文件的属性和物理地址取到内存中，以便后面对文件的快速操作。

书中给出了一段拷贝文件的源码，我觉得讲的很详细，利于我对文件操作的理解。这里我仅截取部分代码。

从代码中可以看出

  1. open()需要文件名和文件读取方式作为参数，返回值是一个文件描述符in_fd，如果打开失败，则返回一个负数。关于文件描述符，书中给出的定义是：value to identify a file。
  2. read()需要三个参数传入，分别是要读取文件的文件描述符，缓冲区指针，要读取数据的长度（这里就是一整个缓冲区的长度）。返回值是读取的字节数rd_count。
  3. write()需要三个参数传入，分别是要写入文件的文件描述符，缓冲区指针，要写入数据的长度。返回写入数据的长度。
  4. close()只需传入需要关闭的文件描述符即可。
